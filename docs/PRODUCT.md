# Product — Bardak

## Суть

Telegram-бот + веб-интерфейс, который превращает хаотичное «Избранное» (или приватный канал) в структурированное личное цифровое пространство.

Пользователь продолжает жить в Telegram — кидает всё подряд (текст, голосовые, ссылки, репосты, фото, PDF, чеки, заметки). Бот автоматически разбирает: принудительно тегирует, даёт поиск, создаёт редактируемые «страницы», позволяет одним кликом шерить подборки.

Основное взаимодействие — внутри Telegram. Веб — для просмотра, редактирования и шеринга. Никакого отдельного приложения.

Активные пользователи Telegram 18–35 лет:

## Функционал

### MVP (первая версия)

**Ядро: сохранить → структурировать → найти → поделиться.**

- **Приём контента** — текст, ссылки, фото, документы/PDF, голосовые, пересланные сообщения. Бот парсит и сохраняет.
- **Принудительное тегирование** — при сохранении бот просит выбрать тег. Если пользователь пропускает — контент уходит в «несортированное» с периодическими напоминаниями.
- **Отложенная сортировка** — «у тебя 12 материалов без тега — разберём?» Бот показывает по одному, пользователь тегирует свайпами/кнопками.
- **AI-описание изображений** — 3-5 слов для поиска. Если AI не уверен — спрашивает пользователя.
- **Расшифровка голосовых** — аудио → текст.
- **Поиск** — по тегам, тексту, типу контента.
- **Страницы** — подборка из нескольких материалов (по тегу или ручной выбор). Редактируемый заголовок и описание.
- **Шеринг** — публичная ссылка на страницу или тег. Просмотр без авторизации.
- **Веб-интерфейс** — лента материалов, просмотр/редактирование страниц, управление тегами.
- **Базовая геймификация** — напоминания о несортированном, прогресс «порядок в бардаке».

### После MVP

- **Память бота** — «запомни, когда я составляю список покупок, добавляй гранатовое вино».
- **«Душа бота»** — кастомный стиль общения.
- **Граф-карта тегов** — визуализация связей между тегами (как в Obsidian, но проще).
- **AI-преобразования** — перевод, переписывание, создание плана из сообщений.
- **Напоминания** — из голосовых и текста (дедлайны, события).
- **Командные сценарии** — группа с ботом, совместные подборки.
- **Голосовой диалог** — как полноценный интерфейс.

## Сценарии использования бота

Три режима, все работают одинаково:

1. **Личный чат с ботом** — основной сценарий. Пользователь пишет боту напрямую.
2. **Приватная группа с ботом** — бот добавлен в группу, обрабатывает все сообщения.
3. **Командный чат** — несколько участников + бот, общие подборки (после MVP).

## Приватность

Архитектурное решение, влияющее на всю разработку:

- Бот хранит **минимум данных**: Telegram user ID, message ID, теги, метаданные. Контент по возможности остаётся в Telegram (file_id).
- **Анонимный режим** — опция, при которой бот не хранит текст сообщений, только ссылки на них.
- Никакого внешнего облачного хранилища для файлов пользователя — используются Telegram file_id или Convex file storage для превью.

## USP (чем отличаемся)

- **Остаёшься в Telegram** — нулевой порог входа, не нужно ставить новое приложение.
- **Принудительное тегирование + геймификация** — бот реально борется с бардаком, а не просто предлагает.
- **Страницы + шеринг** — одна ссылка на красивую подборку. Конкуренты дают максимум экспорт.
- **Приватность** — только ID, минимум данных на серверах.
- **One-time payment** — модель, которую предпочитают в России. Конкуренты — подписка или freemium с жёсткими лимитами.

## MVP-scope (итого)

**Входит:**

- Чат с ботом как основной сценарий
- Приём: текст, ссылки, фото, документы, пересланные
- Обязательный тег или отложенная сортировка
- Поиск по тегам, тексту, типу
- AI-описание изображений
- Расшифровка голосовых
- Страницы (подборки) + шеринг ссылкой
- Базовая геймификация несортированного
- Веб: просмотр, редактирование страниц, управление тегами

**Не входит:**

- Память бота
- Душа / кастомизация личности
- Сложные AI-преобразования
- Граф тегов
- Командные сценарии
- Сложная тарифная матрица
- Голосовой диалог как интерфейс
