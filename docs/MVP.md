# MVP Plan — Bardak (Bot-first)

Этот документ — рабочий план реализации MVP, по которому идём шаг за шагом вместе с ИИ. Фокус: бот как основной продукт + минимальный web только для публикации страниц и их шеринга.

---

## 1) Цель MVP

MVP готов, когда пользователь может:

1. Отправить контент в личный чат с ботом и не потерять его.
2. Разложить контент по тегам сразу или позже через Inbox.
3. Найти сохранённое через бота (по тегу/типу/тексту).
4. Собрать страницу-подборку и получить публичную ссылку.
5. Открыть эту ссылку в web без авторизации.

---

## 2) Scope: что делаем сейчас

### Входит в MVP

- Личный чат с ботом.
- Контент: текст, ссылка, фото, документ, голосовое, пересланное.
- Теги: CRUD + назначение.
- Inbox для элементов без тега.
- Поиск в боте.
- Создание страниц в боте.
- Публичная web-страница для просмотра/шеринга.

### Не входит в MVP

- Полноценный web-кабинет (лента, сложные настройки, полный редактор).
- Командные чаты.
- AI-фичи как обязательная часть core-flow.
- Монетизация и тарифы.

---

## 3) Технические принципы (чтобы не переписывать потом)

- Bot-first архитектура: все ключевые use-case работают в боте без зависимости от web UI.
- Web как thin-view слой: web только читает публичные данные страницы и рендерит их.
- Convex как единый backend: один источник правды для users/content/tags/pages.
- Idempotency ingestion: повторный апдейт Telegram не создаёт дубль.
- Статусы вместо неявной логики: `inbox`, `tagged`, `archived`.
- Все query через индексы, без `filter()` по большим выборкам.
- AI и платёжки подключаются как отдельные слои после MVP, без изменения core-схем.

---

## 4) Пошаговый план (execution)

Текущий статус по ветке `bot-init`:

- Сделан базовый каркас бота (`grammy` + `Elysia`) и long-polling runtime.
- Реализованы `/start` и `/help`, плюс базовый `message:text` flow.
- Подключена запись в Convex для регистрации/обновления `users`.
- Полный ingestion контента (ссылки/фото/документы/голосовые/forwarded) ещё впереди.

## Этап 0. Контракты и схема данных

- [ ] Утвердить минимальные сущности: `users`, `content`, `tags`, `content_tags`, `pages`, `page_items`.
- [ ] Для `content` зафиксировать поля:
  - `source` (`telegram`);
  - `sourceMessageId`, `sourceChatId`;
  - `contentType`;
  - `text` (опционально);
  - `payloadJson`;
  - `status` (`inbox | tagged | archived`).
- [ ] Добавить индексы под:
  - idempotency (`sourceChatId + sourceMessageId + userId`);
  - Inbox (`userId + status`);
  - поиск по типу/дате.

DoD:

- [ ] Схема покрывает все MVP use-case без временных таблиц.
- [ ] Для каждого bot-сценария есть явная модель данных.

## Этап 1. Надёжный ingestion в боте

- [x] `/start`: регистрация пользователя в Convex.
- [ ] Обработка всех MVP-типов контента.
- [ ] Нормализация payload по типам.
- [ ] Idempotent-сохранение контента.
- [ ] Единый ответ пользователю при успешном сохранении.

DoD:

- [ ] Один и тот же Telegram message при ретрае не дублируется.
- [ ] Каждый поддерживаемый тип сообщения сохраняется корректно.

## Этап 2. Теги и Inbox (ядро ценности)

- [ ] CRUD тегов из бота.
- [ ] Назначение тега материалу из бота.
- [ ] Flow «сохранить в Inbox, разобрать позже».
- [ ] Команда разбора Inbox по одному элементу.
- [ ] Команда показать количество непроставленных тегов.

DoD:

- [ ] Любой материал имеет предсказуемый статус.
- [ ] Пользователь может дойти от «хаоса» до «размечено» только через бот.

## Этап 3. Поиск в боте

- [ ] Команда поиска по тегу.
- [ ] Команда поиска по типу.
- [ ] Команда текстового поиска.
- [ ] Лимиты и пагинация выдачи.

DoD:

- [ ] Поиск работает быстро на индексах.
- [ ] Выдача детерминирована и воспроизводима.

## Этап 4. Страницы из бота

- [ ] Создание страницы из выбранных материалов.
- [ ] Редактирование названия и описания страницы в боте.
- [ ] Добавление/удаление материалов из страницы.
- [ ] Генерация публичного `shareSlug`.

DoD:

- [ ] Пользователь может собрать подборку и получить рабочую ссылку.
- [ ] Страница открывается без авторизации.

## Этап 5. Минимальная web-интеграция для шеринга

- [ ] Публичный роут страницы по `shareSlug`.
- [ ] SSR/стабильная загрузка публичной страницы.
- [ ] Рендер контента по типам (текст, ссылка, фото, документ, голосовое).
- [ ] Понятный fallback для удалённого/недоступного контента.

DoD:

- [ ] Ссылка из бота открывается в браузере и показывает подборку.
- [ ] Для просмотра не требуется логин.

## Этап 6. Стабилизация перед расширением

- [ ] `pino` логирование в боте и web.
- [ ] Sentry для ошибок рантайма.
- [ ] Базовый runbook в `docs/`:
  - как проверить webhook/long-polling;
  - как проверить Convex-мутации;
  - как диагностировать 5 самых частых фейлов.

DoD:

- [ ] Ошибки наблюдаемы.
- [ ] Инциденты воспроизводимы и чинятся по инструкции.
